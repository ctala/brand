## ✅ Checklist de Calidad de Contenido (Cristian Tala)

Antes de guardar en NocoDB o calendarizar, verificar:

### Contenido
- [ ] **Test de autenticidad:** ¿Podría haberlo escrito cualquier IA genérica? → Si sí: reescribir
- [ ] **Dato concreto:** ¿Tiene al menos 1 número exacto (no vago)?
- [ ] **Fuente verificada:** ¿Todo dato factual tiene fuente o es experiencia propia verificada?
- [ ] **Voz correcta:** ¿Suena a Cristian o a copywriter genérico?
- [ ] **CTA a Skool:** ¿Post educativo tiene CTA hacia comunidad?
- [ ] **USD siempre:** ¿Ningún monto en CLP?
- [ ] **No hay referencias falsas:** ¿Ningún dato inventado o no verificado?

### NocoDB — Columnas correctas
- [ ] **Content:** Texto limpio publicable. Sin markdown (##, **). Sin notas internas. Tal como aparece en la red social.
- [ ] **Content Preview:** Resumen 1 línea del tema/ángulo del post
- [ ] **Notes / Reviewer Notes:** Para notas internas, contexto, sugerencias al revisor
- [ ] **Asset URL:** URL de la imagen en R2 (verificar que existe y carga)
- [ ] **Has Media:** `true` si tiene imagen/PDF adjunto

### Media
- [ ] **Imagen existe:** La URL de Asset URL responde con 200
- [ ] **Dimensiones correctas:** Según plataforma (ver tabla arriba)
- [ ] **Tamaño mínimo:** >100KB (imágenes muy pequeñas indican error)
- [ ] **Texto en imagen legible:** Si hay texto, verificar que es visible
- [ ] **Paleta correcta:** Synthwave (verde/cyan/magenta sobre negro)
- [ ] **Logo/avatar no regenerado:** Usar solo los assets permanentes
- [ ] **Branding en imagen = URL:** `cristiantala.com`, no `@cristiantala` ni ningún handle
